# Voting / Polling System

A dynamic voting system where admins create polls with custom questions and multiple options (e.g., Class Representative election, Elective Subject selection). Students vote on open polls and see live results. Double-voting is prevented at the database level.

## Proposed Changes

---

### Database (run in Supabase SQL Editor)

#### 3 New Tables + RLS Policies

```sql
-- 1. polls: master poll record, created by admin
CREATE TABLE public.polls (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title       TEXT NOT NULL,
  description TEXT,
  status      TEXT NOT NULL DEFAULT 'open'  -- 'open' | 'closed'
              CHECK (status IN ('open', 'closed')),
  created_by  UUID REFERENCES auth.users(id),
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 2. poll_options: each option that can be voted for
CREATE TABLE public.poll_options (
  id       UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  poll_id  UUID NOT NULL REFERENCES public.polls(id) ON DELETE CASCADE,
  label    TEXT NOT NULL
);

-- 3. poll_votes: one row per user per poll (enforced by UNIQUE constraint)
CREATE TABLE public.poll_votes (
  id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  poll_id   UUID NOT NULL REFERENCES public.polls(id) ON DELETE CASCADE,
  option_id UUID NOT NULL REFERENCES public.poll_options(id) ON DELETE CASCADE,
  voter_id  UUID NOT NULL REFERENCES auth.users(id),
  voted_at  TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (poll_id, voter_id)   -- prevents double-voting
);

-- Enable realtime
ALTER PUBLICATION supabase_realtime ADD TABLE public.poll_votes;

-- ── RLS Policies ─────────────────────────────────────────────────

ALTER TABLE public.polls       ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.poll_options ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.poll_votes  ENABLE ROW LEVEL SECURITY;

-- polls: all authenticated users can read
CREATE POLICY "Read polls" ON public.polls FOR SELECT TO authenticated USING (true);
-- polls: only admin can insert/update/delete (use service role or rely on auth check in code)
CREATE POLICY "Admin manage polls" ON public.polls FOR ALL TO authenticated
  USING (true) WITH CHECK (true);  -- restrict further if you have admin role column

-- poll_options: all authenticated can read
CREATE POLICY "Read options" ON public.poll_options FOR SELECT TO authenticated USING (true);
-- poll_options: authenticated can insert (admin only in practice via UI)
CREATE POLICY "Admin manage options" ON public.poll_options FOR ALL TO authenticated
  USING (true) WITH CHECK (true);

-- poll_votes: users can only read their own vote + all votes for counting
CREATE POLICY "Read votes" ON public.poll_votes FOR SELECT TO authenticated USING (true);
-- poll_votes: authenticated user can insert only their own vote
CREATE POLICY "Cast own vote" ON public.poll_votes FOR INSERT TO authenticated
  WITH CHECK (voter_id = auth.uid());
-- No UPDATE or DELETE on votes (immutable)
```

- Fields: Title (`text`), Description (`textarea`), Options (dynamic — "Add Option" button appends input rows)

#### JS functions added inside the existing `<script type="module">`
| Function | What it does |
|---|---|
| `loadPolls()` | Fetches all polls + counts votes per option via JS aggregation |
| `openCreatePollModal()` | Resets and shows the create-poll modal |
| `addOptionField()` | Dynamically appends an option input row |
| `submitCreatePoll()` | Inserts poll then bulk-inserts options |
| `togglePollStatus(id, status)` | Updates `polls.status` to open/closed |
| `deletePoll(id)` | Deletes poll (cascade deletes options + votes) |
| `showPollResults(pollId)` | Inline expander showing per-option vote counts |
